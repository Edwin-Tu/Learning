# Linux 系統資訊查詢

在 Linux 系統中，了解如何獲取系統資訊是一項基本但重要的技能。無論是日常系統管理、問題排解，還是安全評估，這些知識都是不可或缺的。

## 為什麼需要了解系統資訊？

- 系統管理：了解系統狀態、配置和資源使用情況
- 故障排除：診斷系統問題和效能瓶頸
- 安全評估：識別潛在的安全漏洞和配置錯誤
- 系統稽核：確認系統配置符合預期

## 常用系統資訊命令

| 命令 | 描述 | 常見用途 |
|---|---|---|
| `whoami` | 顯示目前使用者名稱 | 確認當前執行身份 |
| `id` | 顯示使用者身份和群組資訊 | 檢查權限和群組成員身份 |
| `hostname` | 顯示或設定系統主機名稱 | 識別系統身份 |
| `uname` | 顯示系統資訊 | 獲取核心與作業系統資訊 |
| `pwd` | 顯示目前工作目錄 | 確認目前位置 |
| `ifconfig`/`ip` | 顯示網路介面資訊 | 檢查網路配置 |
| `netstat`/`ss` | 顯示網路連線狀態 | 監控網路活動 |
| `ps` | 顯示執行中的程序 | 監控系統程序 |
| `who` | 顯示已登入使用者 | 檢查系統使用狀況 |
| `env` | 顯示環境變數 | 檢查系統環境設定 |
| `lsblk` | 列出區塊裝置 | 檢查儲存裝置 |
| `lsusb` | 列出 USB 裝置 | 檢查 USB 裝置 |
| `lsof` | 列出開啟的檔案 | 檢查檔案使用情況 |
| `lspci` | 列出 PCI 裝置 | 檢查硬體資訊 |

## 詳細命令說明與範例

### 1. 使用者資訊查詢

#### whoami - 查看目前使用者
```bash
$ whoami
edwin
```

#### id - 查看使用者與群組資訊
```bash
$ id
uid=1000(edwin) gid=1000(edwin) groups=1000(edwin),4(adm),24(cdrom),27(sudo)
```

> 🔍 **重要提示**：`sudo` 群組成員可以執行特權命令，這在安全評估時需特別注意。

### 2. 系統資訊查詢

#### hostname - 系統名稱
```bash
$ hostname
nixfund
```

#### uname - 系統詳細資訊
```bash
# 顯示所有系統資訊
$ uname -a
Linux box 4.15.0-99-generic #100-Ubuntu SMP Wed Apr 22 20:32:56 UTC 2020 x86_64 x86_64 x86_64 GNU/Linux

# 只顯示核心版本
$ uname -r
4.15.0-99-generic
```

> ⚠️ **安全提示**：核心版本資訊可能揭示系統是否有已知漏洞，建議定期更新系統。

## SSH 遠端連線

SSH（Secure Shell）是安全地遠端存取 Linux 系統的標準方式：

```bash
# SSH 連線語法
ssh username@hostname

# 範例
ssh htb-student@10.10.10.10
```

SSH 的特點：
- 加密通訊確保安全性
- 無需圖形介面，資源使用效率高
- 支援金鑰認證，增強安全性
- 廣泛用於系統管理和維護

## 學習建議

1. **循序漸進**
   - 從基本命令開始
   - 逐步探索更多選項
   - 實際操作加深理解

2. **善用手冊頁面**
   ```bash
   man uname    # 查看 uname 的完整手冊
   uname --help # 快速查看選項說明
   ```

3. **建立學習環境**
   - 使用虛擬機練習
   - 記錄常用命令
   - 模擬實際場景

4. **保持探索精神**
   - 了解命令的不同選項
   - 嘗試組合不同命令
   - 觀察系統行為

> 💡 **學習提示**：不要害怕犯錯！每個系統管理員都是從基礎開始學習的。重要的是持續練習和探索。

## 安全性考量

1. **資訊收集**
   - 定期檢查系統狀態
   - 注意異常的進程或連線
   - 監控系統資源使用

2. **權限管理**
   - 經常檢查使用者權限
   - 注意特權群組成員
   - 定期稽核系統存取

3. **系統更新**
   - 追蹤核心版本
   - 及時套用安全更新
   - 監控已知漏洞
